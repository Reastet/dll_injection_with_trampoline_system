# dll_injection_with_trampoline_system
Демонстрационный проект, использующий механизм dll инъекции для перехвата вызова kernel32 функций. Исключительно учебный проект

Подгружает dll в память процесса, путем выделения доп. памяти в целевом процессе, переноса строки-пути до dll и запуска LoadLibrary, после чего dll начинает работать в режиме мониторинга либо перехвата

new_dll_injector является разработанной dll. Перед сборкой необходимо вручную собрать .asm файл для сохранения контекста при вызове функции для мониторинга

needle-управляющая программа, выполняющая встраивание dll и выводящая информацию в консоль
Список аргументов:
| № | флаг | описание |
| ------ | ------- | ------|
|1 | –pid <PID целевого процесса>| Целевой процесс выбирается по значению PID |
|2| –name < Имя целевого процесса> | Целевой процесс выбирается по имени|
|3 |–func <Имя функции> | Имя функции, вызов которой требуется отследить|
|4 |–hide <Имя файла> |Имя файла, который надо скрыть от целе|
